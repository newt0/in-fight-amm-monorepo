
📌 1. 奖池与市场预测总览

位于预测区域顶部，用于展示当前整个市场的总体状态。

元素：

奖池总金额：如 1,000 USDC

市场预测占比条（A vs B）

以横向条形图形式呈现，如：

选手 A：62%

选手 B：38%

文案示例：
当前市场预测：A 62% / B 38%

特点：

实时更新

数值变化以颜色提示（上涨绿色 / 下跌红色）

📌 2. 选手卡片（A / B 通用结构）

预测市场 UI 中每位选手都有独立卡片，包含当前价格、图表、操作等内容。以下为统一结构。

### 2.1 头部信息

选手名称（如 “选手 A”）

当前票价（例如 100 USDC / Ticket）

若用户持仓：

你持有：10 Tickets（成本 1,000 USDC）

### 2.2 操作模式切换

两种模式通过顶部的切换按钮切换。

[ Buy ]   [ Sell ]


选中的为深紫色

未选中的为白底黑边框

切换后底部主按钮文案同步变化

Buy → “确认买入”

Sell → “确认卖出”

### 2.3 价格趋势图(要用trading view)

位于卡片中段，用于展示该选手价格的实时变化。

规格：

高度约 80–100px

浅紫色背景

右上角可切换时间区间：

1m / 5m / 实时

实时更新价格线

### 2.4 数量与总额（同一行）
[-]  10  [+]   Tickets       |       总额：1,000 USDC


左：数量选择器（可加减，也可直接输入）

右：根据当前数量与价格自动计算 Total

### 2.5 余额显示

可用余额：1,000 USDC

或在 Sell 模式中显示：

可卖出：10 Tickets

### 2.6 主操作按钮（全宽）
[ 确认买入 ]
或
[ 确认卖出 ]


深紫底白字

点击后调用买卖逻辑成交

📌 3. 选手 B 卡片

结构与选手 A 完全一致，只替换对应数据：

示例：

当前价格：80 USDC / Ticket

用户持仓：5 Tickets（价值 400 USDC）

初始模式自动根据持仓选择 Sell（若持仓>0）

📌 4. 我的账户（底部区域）

用于展示用户整体资产情况。

余额：
Balance：1,000 USDC

持仓表格：
Holding	Tickets	Market Value
选手 A	10	1,000 USDC
选手 B	5	400 USDC

Market Value 根据当前市场价格动态计算

不显示历史成本（减少视觉负担）
